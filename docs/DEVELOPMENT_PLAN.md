создай# План разработки RLM (Development Plan)

## 1. Общая информация

**Цель проекта:** Создать инструмент, позволяющий агроному просматривать спутниковые снимки полей в различных спектрах (RGB, False Color, NDVI, NDWI и др.), с автоматическим наложением контура поля и возможностью LLM-анализа.

**Ключевые требования:**
- Буферная зона 500 метров от границы поля (выносится в параметр).
- Поиск малооблачных сцен Sentinel-2 Level-2A.
- Выбор даты и типа продукта.
- Визуализация с наложением векторного контура поля.
- Интеграция с MCP и Qwen3 (OpenRouter).

**Тесты и проверка корректности:**

- **Алгоритмические тесты:** Сравнение рассчитанных индексов с эталонными значениями на синтетических данных (например, NDVI должен быть в диапазоне [-1, 1]).
- **Как проверить:** Запустить тесты `pytest tests/test_processor.py`. Проверить, что после обработки растра значения NDVI соответствуют ожидаемым для вегетации (0.4–0.9).
- **Пользовательское тестирование:** Возможно после завершения Этапа 2. Агроном может проверить корректность маски облачности и индексов на реальных полях.

## 3. Технологический стек (обновлённый)

- **Язык:** Python 3.10+

- **Конфигурация:** `pydantic-settings`, `.env`
